# SorteringsAlgoritmer
This repository contains five sorting implementations: a basic MergeSort and an optimized version using insertion sort for small subarrays, the same pair for QuickSort, and a TreeMap-based implementation using a Red-Black tree, where keys are elements and values store their frequencies.

Detailed;
This repository contains five algorithm implementations and a testfile to test execution time to compare algorithms. It includes a basic implementation of MergeSort and an optimized version that switches to insertion sort for smaller subarrays. The same approach has been applied to QuickSort, with both a standard and an optimized version.

However, it is important to note that the optimized version of QuickSort did not show a significant improvement in execution time compared to the optimized MergeSort implementation.

Additionally, the repository includes a sorting implementation based on a symbol table. This version uses a Red-Black binary search tree (TreeMap) to sort the list by inserting elements as keys and storing their occurrence counts as values.
